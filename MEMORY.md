## 📚 关键决策

### 决策 1: 为什么选择轻量化设计？
**时间**: 2026-05-09  
**决策**: 最小化文件数量（AGENTS.md + MEMORY.md），而不是创建 SOUL.md、PROTOCOLS.md 等  
**原因**:
- 信息不重复：每条信息有唯一来源
- 易于维护：新对话时加载成本低
- 可扩展：需要时再拆分（如 WORKFLOWS.md）
- 符合 GTD 原则："单一事实来源"

### 决策 2: 为什么分离长期记忆 vs 短期记录？
**时间**: 2026-05-09  
**决策**: MEMORY.md（不变信息）vs /records（日志）  
**原因**:
- 查询效率：重要信息集中，不被日志淹没
- 更新频率不同：MEMORY.md 月度，/records 日更新
- 噪音消除：临时想法不污染长期认知

### 决策 3: 为什么用 GitHub 而不是本地文件？
**时间**: 2026-05-09  
**决策**: 将仓库作为持久化文件空间，所有信息版本化存储  
**原因**:
- 可追溯：历史记录完整，可回顾决策演变
- 可分享：需要时可与他人共享
- 可同步：多设备、多会话无缝继续
- Git 友好：所有内容都是文本，易于 diff 和 merge

## 💡 系统架构理念

```
┌─────────────────────────────────────────┐
│ 新对话启动                              │
│ → 加载 AGENTS.md（了解角色）            │
│ → 加载 MEMORY.md（了解背景）            │
│ → 查看 /records 最新记录（了解进度）    │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ 执行任务                                │
│ → 思考 → 代码/文档 → 测试               │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ 收尾动作                                │
│ 1. 代码/文档 → /code 或 /docs           │
│ 2. MEMORY.md → 添加新决策/学习          │
│ 3. /records/{date}.md → 记录日志        │
│ 4. Git commit → 同步到仓库              │
└─────────────────────────────────────────┘
```
