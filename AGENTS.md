## 🔄 工作循环

### 每次对话启动
1. **加载上下文** → 检查 `MEMORY.md` 和 `/records` 目录中的最新记录
2. **确认状态** → 了解当前项目进度、待办任务、角色上下文
3. **保持一致性** → 使用之前定义的角色、术语、工作风格

### 完成任务后的收尾
```
✅ 任务完成 → 立即执行这些动作：
  1. 更新 MEMORY.md（添加新的发现、决策、重要信息）
  2. 在 /records/{日期}.md 中记录工作日志
  3. 如生成新代码/文档，放入 /docs、/code、/templates 等对应目录
  4. 同步到 main 分支（通过 commit 或 PR）
  5. 在对话中说明"✓ 已保存到仓库"
```

## 💾 记忆管理规则

### 长期记忆 (MEMORY.md)
- **不变的东西**：角色、目标、原则、架构决策
- **关键决策记录**：为什么选择某个方案
- **项目元数据**：关键技能、工具、依赖、集成
- **定期回顾**：每月检查一次，删除过时信息

### 短期记录 (/records)
- **日志格式**：`YYYY-MM-DD.md`（一天一个文件）
- **记录内容**：
  - 当日完成的任务
  - 遇到的问题与解决方案
  - 新的想法或改进方向
  - 待办项（carry-over to next day）

## 🚀 使用场景示例

### 场景 1: 新对话，询问"我在做什么？"
```
→ 检查 MEMORY.md 和 /records 最新记录
→ 总结当前状态和上一步进展
→ 询问："从XX继续？还是新任务？"
```

### 场景 2: 完成代码实现
```
→ 代码写完，测试验证
→ 将代码保存到 /code（带说明注释）
→ 更新 MEMORY.md（如涉及架构决策）
→ 新增 /records/{date}.md 记录
→ 同步到仓库（push commit）
→ 说明："✓ 代码已保存到 /code/{filename}，日志见 /records/{date}.md"
```

### 场景 3: 积累知识/工具
```
→ 发现有用的工具、模板、框架
→ 保存到 /templates 或 /docs
→ 更新 MEMORY.md 的"可用工具"部分
→ 下次需要类似任务时可直接复用
```

## 🔧 扩展性

这套系统是**最小化设计**，可按需扩展：

- **需要更多细节？** → 在 MEMORY.md 中添加新章节
- **项目变复杂？** → 在 /projects 中创建子目录
- **需要工作流？** → 添加 WORKFLOWS.md
- **需要技能？** → 在 /skills 中维护能力清单

**原则**：先有简单版本，再根据实际需求迭代。
